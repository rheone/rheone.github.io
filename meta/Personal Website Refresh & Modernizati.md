# Personal Website Refresh & Modernization

This repository contains the personal website and portfolio for Robert H. Engelhardt, deployed via GitHub Pages at:

[rheone.github.io](https://rheone.github.io/?utm_source=chatgpt.com)

The site is currently built using plain HTML, CSS, and JavaScript with no frameworks or build tooling. The goal of this effort is to modernize the presentation and improve the overall professionalism, usability, and visual consistency of the site while preserving and improving the existing content.

## Technical Requirements

* Continue using a static-site approach with vanilla HTML, CSS, and JavaScript.
* Do not introduce build systems, bundlers, transpilers, or frontend frameworks.
* Prefer CDN-hosted assets and libraries whenever practical.
* Use:

  * Bootstrap for layout and responsive presentation
  * Font Awesome for icons
* Consolidate styling into shared CSS files wherever possible.

## Design & Presentation Goals

Create a modern, polished, professional presentation suitable for a software developer portfolio.

### Visual Style

* Clean and unobtrusive
* Minimalist and modern
* Responsive across desktop and mobile devices
* Primarily monochrome:

  * black
  * white
  * shades of gray
* Use restrained accent colors only where they improve usability or visual hierarchy.

### Typography

* The homepage “About Me” section should use a clean monospace font such as:

  * Fira Code
  * JetBrains Mono
  * similar professional developer-oriented monospace fonts

## Content & Layout Expectations

* Rework the overall presentation and layout of pages as needed.
* Do not feel constrained by the current page structure or formatting.
* Preserve all meaningful content.
* Improve:

  * grammar

  * readability
  * organization
  * visual hierarchy

### Navigation

* Add a clean, professional top navigation/menu.
* Ensure navigation is consistent across pages.
* Include clear links to major sections/pages.

### Footer

## Quality Expectations

* Be critical and detail-oriented.
* Double-check work for:

  * consistency
  * broken links
  * accessibility
  * responsive layout issues
  * spelling/grammar errors
  * visual polish
* Avoid unnecessary visual clutter or gimmicks.
* The final result should feel cohesive, modern, and professionally designed.

## Important Constraints

* Do not remove meaningful content unless it is clearly obsolete or duplicated.
* Content should be preserved and improved — not discarded.
* Maintain fast load times and simplicity appropriate for a static GitHub Pages site.

### Accessibility & Standards

* Follow modern accessibility best practices (WCAG-aware where practical).
* Ensure:

  * sufficient color contrast
  * keyboard navigability
  * semantic HTML usage
  * proper heading hierarchy
  * alt text for meaningful images/icons
* Avoid accessibility regressions introduced by visual redesigns.

### Performance

* Minimize unnecessary JavaScript.
* Optimize for fast GitHub Pages load times.
* Use lazy-loading for images where appropriate.

### Dark Mode

* Add support for dark mode, preferably using:

  * CSS variables
  * `prefers-color-scheme`
* Maintain a professional monochrome aesthetic in both light and dark themes.

### Mobile & Responsive Design

* Verify layouts on:

  * desktop
  * tablet
  * mobile
* Ensure navigation and project cards remain usable on small screens.

### SEO & Metadata

* Improve metadata and discoverability:

  * page titles
  * meta descriptions
  * Open Graph tags
  * favicon support
* Add structured and consistent document titles.

### Developer-Focused Enhancements

* Add tasteful technology badges/icons where appropriate.

### Project Presentation

* Prefer clean card-based layouts over dense text walls.

### Code Quality

* Remove duplicated CSS/JS where possible.
* Standardize spacing, typography, and layout conventions.
* Keep HTML structure maintainable and readable.

### Progressive Enhancement

* The site should remain functional with minimal JavaScript enabled.
* Avoid fragile or over-engineered client-side behavior.

### GitHub Pages Compatibility

* Ensure all paths and assets work correctly under GitHub Pages hosting.
* Avoid assumptions requiring server-side functionality.

### Content Authenticity

* Do not fabricate project details, experience, metrics, or accomplishments.
* If additional information is unavailable, keep descriptions concise rather than speculative.

### Visual Consistency

* Maintain a consistent:

  * spacing system
  * typography scale
  * icon style
  * card/button treatment
  * section hierarchy
* The site should feel cohesive rather than page-by-page inconsistent.

### Final Review Expectations

Before finalizing:

* Validate HTML and CSS for major issues.
* Check for console errors.
* Verify all external links.
* Ensure there are no placeholder assets or lorem ipsum remnants.
* Review pages for alignment and spacing inconsistencies.

## Collaboration

If additional clarification or design direction is needed, ask targeted questions before making major assumptions.
